Overview

Thank you for using WPortfolio for our personal web blog portfolio. WPortfolio suits a person who want to show their works while writing some interesting blog.

This documentation will walk us around about how to use this theme on our own web at no cost.

Requirements

This theme is created by a Developer for Developers, hence it is going to be a bit techy for some of people out there.

Before using this theme, please make sure that we have these requirements:

  • Git Base Knowledge, we will download the theme using git command.
  • WordPress Base Knowledge, before using this theme, we are assuming that we are already familiar with basic WordPress Development.
  • Github Account, a place where we will show our works and highlight them on our website. However, if it is optional if we don't want to show it.

Installation

WPortfolio is available for free and we can download it from its GitHub repository.

  1. Download it from its Github repository 
    git clone https://github.com/rendy44/wportfolio.git
    Once we've done it, we should have two themes, wportfolio as our main theme and wportfolio-child as a demo child theme to modify the parent theme.
  2. Make sure our working directory is correct as root/wp-content/themes/wportfolio/style.css.
  3. Before activating the theme, please define constants named GITHUB_USER with our GitHub username (without any slices) and GITHUB_KEY with a personal access token in our wp-config.php file. It is not necessary to grant all permissions, we only need to grant user:read permission. 
    define( 'GITHUB_USER', 'githubusername' );
define( 'GITHUB_KEY', 'randomstringthatcontainaccesstoken' );
  4. Activate the WPortfolio Child Demo theme from our wp-admin.

Configuration

After Installing the theme, we need to provide two pages as our front-page that will act as our landing page and home-page where we will store our posts.

  1. Go to our wp-admin and click Pages > Add New. We can name it as free as we want, but for easier management, it is suggested to name it Front.
  2. Repeat the above step but name the new page with Posts.
  3. Go to Appearance > Customize > Homepage Settings, in our homepage displays please select Static Page, Select Front in Homepage dropdown and Posts in Posts Page dropdown.

Now, try to open our homepage, we should see our website full of built-in contents.
